# how_to_use_formative-analysis

> 🚧 **The module is under development / pending integration into the paideia repository.**
>
> The code itself is already **complete** in a separate repository (`~/localgit/formative-analysis/`),
> but it has not yet been migrated and integrated into `modules/` of the paideia umbrella workspace.
> Once integrated, this document will be filled in with usage instructions that follow the paideia conventions (Bronze/Silver/Gold and the CLI).

---

## What this module will be

A module that, each week, produces **a formative-assessment test sheet, an analysis of the responses, and per-student reports**.

| Item | Plan |
|---|---|
| Input | Short-answer candidates generated by maieutica + student responses |
| Output | Test-sheet PDF + per-student analysis report |
| Point in the semester | Every week during the semester |
| Dependencies | maieutica (or manual entry) |
| Status | Code complete, awaiting integration into paideia |

## Integration roadmap

According to `idea/paideia-idea.md`, formative-analysis was completed as a separate predecessor project and is to be absorbed into paideia's `modules/`. retro-mester's per-week trajectory enrichment input will also be added after this integration.

## Current alternative

Until integration, you can produce similar output with the following.

- The **formative-test-creator** skill — short-answer formative-assessment items + rubrics + support plans
- The existing tools in the separate repository `~/localgit/formative-analysis/`
